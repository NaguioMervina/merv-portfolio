# Plan: GitHub Action — Daily GitLab Contributions Refresh

## Goal
Auto-refresh GitLab contribution data daily on GitHub's servers. No local action needed.

## What to create
A single GitHub Actions workflow file: `.github/workflows/refresh-gitlab-contributions.yml`

## Workflow details

```yaml
name: Refresh GitLab Contributions

on:
  schedule:
    - cron: '0 8 * * *'  # Daily at 8:00 AM UTC
  workflow_dispatch:       # Allow manual trigger from Actions tab

jobs:
  refresh:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - run: npm run refresh:gitlab
        env:
          GITLAB_USERNAME: NaguioMervina

      - name: Commit and push if changed
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add resources/js/data/gitlab-contributions.json
          git diff --cached --quiet || git commit -m "chore: refresh GitLab contributions [skip ci]" && git push
```

## Key points
- Runs daily at 8 AM UTC (you can change the cron time)
- Manual trigger available via "Run workflow" button in GitHub Actions tab
- Uses `[skip ci]` in commit message to prevent infinite workflow loops
- Only commits if the JSON actually changed (the script already handles this, but the `git diff --cached --quiet` is a safety net)
- No secrets needed — GitLab calendar endpoint is public

## Files to create

## Verification
1. Push the workflow file to GitHub
2. Go to repo → Actions tab → "Refresh GitLab Contributions" → "Run workflow"
3. Wait ~30 seconds, check if a new commit appears with updated contributions JSON
4. Verify the schedule triggers daily by checking the Actions tab after 8 AM UTC
